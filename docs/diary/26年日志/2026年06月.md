**计算机带符号的数据运算是基于补码，那不带符号的数据运算呢？**

```txt
⏺ 无符号数据没有"补码"的概念，它直接按二进制原值运算。

  核心区别：

  - 有符号数：最高位是符号位，负数用补码表示，运算（加减）统一用补码进行。
  - 无符号数：所有位都是数值位，没有符号位，直接参与运算。

  举例（8位）：

  ┌───────────┬────────────┬────────────────────┐
  │  二进制   │ 无符号解释 │ 有符号（补码）解释 │
  ├───────────┼────────────┼────────────────────┤
  │ 0000 0101 │ 5          │ +5                 │
  ├───────────┼────────────┼────────────────────┤
  │ 1111 1011 │ 251        │ -5                 │
  ├───────────┼────────────┼────────────────────┤
  │ 1111 1111 │ 255        │ -1                 │
  └───────────┴────────────┴────────────────────┘

  运算层面的关键点：

  1. 加减法电路是同一套 —— 无符号和有符号用的是同一个加法器，位级结果完全相同，区别只在于如何解读结果以及看哪些标志位。
  2. 进位/借位标志 CF（Carry Flag） —— 无符号运算关注这个，表示是否溢出了表示范围。
  3. 溢出标志 OF（Overflow Flag） —— 有符号运算关注这个，表示结果是否超出了有符号范围。

  所以本质上：硬件不区分有符号和无符号，它只做二进制加法。 是程序员/编译器通过选择读哪个标志位来决定如何解读结果。
```

## PS后期学习记录

**HDR 是什么？**

HDR 是 High Dynamic Range（高动态范围）的缩写。
简单理解，它的核心作用是：让亮的地方更亮、暗的地方更暗，同时保留亮部和暗部的细节，使得画面更接近人眼在真实世界中看到的明暗对比和色彩。

**B&W 是什么？**

在摄影PS后期中，B&W 是 Black and White（黑白）的缩写。它指的并不是一个单一的功能，而是一整套将彩色图像转换为单色调（灰度）照片的处理流程和方法
