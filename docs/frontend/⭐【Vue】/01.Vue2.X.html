  • 一个页面中只能存在一个Vue实例，不能创建多个Vue实例.
  • vue实例中el属性代表vue实例的作用范围，日后再vue实例作用范围内可以使用直接获取data中变量名对应属性值.

  • 使用进行data中数据获取时，可以在取值语法中进行算术运算 逻辑运算以及调用相关类型的相关方法.
  • vue实例中el属性可以书写任何css选择器，但是推荐使用id选择器，因为一个vue实例只能作用于一个具体作用范围.

1.2.1 v-text： ​

v-text：用来获取data中数据将数据以文本的形式渲染到指定标签内部 类似于javascript 中 innerText

html
<div id="app" class="aa">

    <span v-text="message"></span>
</div>

<script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
<script>
    const app = new Vue({
        el:"#app",
        data:{
            message:"百知欢迎您"
        }
    })
</script>

插值表达式 和 v-text 对比：

  • 使用v-text取值会将标签中原有的数据覆盖 使用插值表达式的形式不会覆盖标签原有的数据
  • 使用v-text可以避免在网络环境较差的情况下出现插值闪烁

  • **局部组件：**只能在注册组件中使用组件

<script>
export default {
  name: "LoginForm",

      password: "",

}
</script>

在需要使用当前局部组件的组件中引入当前局部组件：

vue
<script>
import LoginForm from "../components/login/LoginForm.vue";

export default {
  name: "Login",
  // 使用函数式 data，进行数据双向绑定：
  data() {
    return {
      time: "",
      date: "",
    }
  },
  // 使用 Vue 钩子函数，每秒触发 update_clock 函数：
  mounted() {
    this.$nextTick(() => {
      setInterval(this.update_clock, 1000)
    })
  },
  methods: {
    // 日期时间计算函数：
    update_clock: function (e) {
      let d = new Date();
      let year = d.getFullYear();
      let month = d.getMonth();
      month++;  // 月默认是从0开始所以要+1
      if (month < 10) month = "0" + month;
      let day = d.getDay();
      if (day < 10) day = "0" + day;
      let hour = d.getHours();
      if (hour < 10) hour = "0" + hour;
      let minute = d.getMinutes();
      if (minute < 10) minute = "0" + minute;
      let second = d.getSeconds();
      if (second < 10) second = "0" + second
      this.time = hour + ":" + minute + ":" + second;
      this.date = year + "/" + month + "/" + day;
    }
  },

  // 引入局部组件：
  components: {
    loginForm: LoginForm,
  }
}
</script>

4.3 组件属性： ​

data 属性必须使用函数方式，并且使用return关键字进行数据返回。

  • **name：**组件名称，对应是作为标签名在父组件中进行引入
  • **data：**组件维护的双向绑定的数据
  • **computed：**计算属性
  • **methods：**组件函数
  • 生命周期函数
  • **components：**挂载子组件

4.4 组件之间的数据传递 Props 机制： ​

Props 用来给组件传递相应静态数据或者是动态数据，数据只能由父组件向子组件传递数据。

**单向数据流：**所有的 props 中的数据，只能由父组件传递给子组件，在子组件进行操作。反过来则不行，这样做的目的是防止子组件意外改变父组件的状态，从而导致数据的流向难以理解。

每次父组件发生变更时，子组件中所有的 props 都会刷新为最新值。

传递静态数据：

使用组件内部属性 props 接收由父组件传递过来的静态数据。

传递动态数据：

组件标签不使用键值对的方式进行值传递，而是使用 Vue 提供的 : 进行属性的双向绑定，实现数据的传输。

4.5 组件之间的事件传递： ​

可以通过组件的事件传递实现子组件属性传递给父组件。如何实现组件之间的事件传递？传递方式和 props 传递方式相似。

现在父组件中定义事件函数，函数参数是接收子组件传递的参数（可以传递多个）

子组件调用父组件传递的函数，通过this.$emit调用父组件传递的方法，通过参数传递子组件数据，可以传递多个参数。

5.Vue 路由： ​

介绍 | Vue Router

  • 后端路由：请求方式、请求地址与function处理函数之间的对应关系。

  • 前端路由和SPA：

    • **SPA：**指的是Web网站只有唯一的一个HTML页面，所有组件的展示与切换都在这个唯一的一个页面内完成，不同组件之间的切换需要通过前端路由来实现。
    • **前端路由：**Hash地址与组件之间的对应关系。

​ 路由，改变URL后，页面不会整体刷新。路由可以理解为组件间的指向。路由表本质是一个记录指向的关系的映射表，由Key-Value键值对表示，Key：表示路由；Value：可以是function函数或者是component组件。

前端路由的工作方式：

5.2.1 路由匹配： ​

​ 创建一个router路由配置文件：

javascript
// 导入组件：
import Home from "../components/Home.vue"
import Content from "../components/Content.vue";
import {createRouter, createWebHashHistory} from "vue-router";

// 定义路由：
const routes = [
    {path: '/', component: Home},
    {path: '/content', component: Content},
]

// 创建Vue Router 实例：
const router = createRouter({
    // 内部提供了 history 模式的实现。为了简单起见，我们在这里使用 hash 模式。
    history: createWebHashHistory(),
    routes, // `routes: routes` 的缩写
})

// 导出Vue Router 实例：
export default router

​ 在main.js入口中挂在路由：

javascript
import {createApp} from 'vue'
import './style.css'
import App from './App.vue'

// 挂在路由到main.js中：
import router from "./router/index.js";

// main.js 入口文件
const app = createApp(App)

// 挂在路由组件：
app.use(router)
app.mount('#app')

5.2.2 动态路由匹配： ​

​ 带参数的动态路由配置，路径参数 用冒号 : 表示。当一个路由被匹配时，它的 params 的值将在每个组件中以 this.$route.params 的形式暴露出来。

javascript
// 这些都会传递给 `createRouter`
const routes = [
  // 动态字段以冒号开始
  { path: '/users/:id', component: User },
]
javascript
const User = {
  template: '<div>User {{ $route.params.id }}</div>',
}

​ this.$route：当前活跃的路由，也就是当前路由。

获取动态路由参数：

  • 使用$route.params 参数对象，可以直接访问到动态匹配的参数指。
  • 使用props接收路由参数：

为了简化路由参数获取形式，vue-router允许路由规则中开启props传参

JavaScript
// 1.定义路由规则时，声明 props: true 可以在组件中，以props的形式接收到路由规则匹配到的参数
{path: '/login/:id', component: Login, props: true}
vue
<script>
export default {
  // 2.在对应的组件中使用props，接收路由参数：
  props: ['id']
}
</script>

// 3.在组件中可以获取
<template>
  <div>
    {{id}}
  </div>
</template>

5.2.3 路由重定向： ​

​ 路由重定向指：用户在访问地址A的时候，强制用户跳转到地址C，从而展示特定的组件页面。通过路由规则的redirect属性，指定一个新的路由地址，可以很方便的设置路由的重定向。

javascript
import { createRouter, createWebHistory } from 'vue-router'

// 配置路由：
const routes = [
  {
    path: '/home',
    name: 'home',
    component: () => import('../views/HomeView')
  },
  {
    // 设置一个路由重定向：
    path: '/redirect',
    redirect: '/home'
  }
]

// 创建路由对象：
const router = createRouter({
  history: createWebHistory(process.env.BASE_URL),
  routes
})

export default router
