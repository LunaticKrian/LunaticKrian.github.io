1.3 Lombok注解： ​

  • @Data：用在类上，生成实体类的 get、set、toString、hashCode、equals方法

  • @Getter：用在类上，生成 get 方法

  • @Setter：用在类上，生成 set 方法

  • @ToString：用在类上，生成 toString 方法

  • @AllArgsConstructor：用在类上，生成全参构造方法

  • @NoArgsConstructor：用在类上，生成无参构造方法

  • @Accessors(chain = true)：给实体类中的 set 方法开启链式调用；这个注解有一个属性值 chain，true：开启链式调用，false：关闭链式调用。

  • @Slf4j：用在类上，用来快速给类定义一个日志变量 log，log 支持使用 {} 进行占位参数拼接

    java
    private final Logger log = LoggerFactory.getLogger(this.getClass());

3.1 参数校验概述： ​

Jakarta Bean Validation - Home

The Bean Validation reference implementation. - Hibernate Validator

Bean Validation 提供了参数校验的规范，Hibernate Validator 提供了具体的参数校验规则的实现。

Java EE规范：

​ 以javax开头的包，这些包只是提供了很多API（接口），不由JDK来实现，而是由具体的厂商进行实现。

​ 并不是只有这些，还有一些JDK并不自带的JavaEE规范，这些规范需要引入第三方的依赖。

规范的制定：

  • JCP：The Java Community Process(SM) Program (jcp.org)

  • JSR：Java规范提案

javax 会逐渐转向 Jakarta（开源）

构建工具类：

java
import javax.validation.ConstraintViolation;
import javax.validation.Validation;
import javax.validation.Validator;
import java.util.List;
import java.util.Set;
import java.util.stream.Collectors;

public class ValidationUtils {
    // 线程安全：
    private static Validator validator;
    static {
        validator = Validation.buildDefaultValidatorFactory().getValidator();
    }

    /**
     * 参数校验方法
     *
     * @param t 被校验的参数
     * @return List 校验信息
     * @param <T> 传入参数指定泛型
     */
    public static <T> List<String> valid(T t) {
        // 进行参数校验，如果检验没通过，返回的 validate 中，保存了校验信息：
        Set<ConstraintViolation<T>> validate = validator.validate(t);
        List<String> list = validate.stream().map(v -> "属性：" + v.getPropertyPath()
                        + "\t属性值：" + v.getInvalidValue()
                        + "\t提示信息：" + v.getMessage())
                .collect(Collectors.toList());

        return list;
    }
}

​ 需要配置注解进行使用：

java
public class User {

    @NotNull
    private String name;
    private int age;

    @Test
    public void test() {
        User user = new User();
        List<String> valid = ValidationUtils.valid(user);
        System.out.println(Arrays.toString(valid.toArray()));
    }
}

3.4 常用的校验参数注解： ​

javax.validation.constraints

3.4.1 空和非空检查： ​
  • @NotBlank：只能用于字符串不为 null ，并且字符串 .trim() 以后 length 要大于 0 。
  • @NotEmpty：集合对象的元素不为 0 ，即集合不为空 。
  • @NotNull：不能为 null 。
  • @Null：必须为 null 。
3.4.2 数值检查： ​
  • @DecimalMax(value)：被注释的元素必须是一个数字，其值必须小于等于指定的最大值。
  • @DecimalMin(value)：被注释的元素必须是一个数字，其值必须大于等于指定的最小值。
  • @Digits(integer, fraction)：被注释的元素必须是一个数字，其值必须在可接受的范围内。
  • @Positive：判断正数。
  • @PositiveOrZero：判断正数或 0 。
  • @Max(value)：该字段的值只能小于或等于该值。
  • @Min(value)：该字段的值只能大于或等于该值。
  • @Negative：判断负数。
  • @NegativeOrZero：判断负数或 0 。
3.4.3 Boolean 值检查： ​
  • @AssertFalse：被注释的元素必须为 true 。
  • @AssertTrue：被注释的元素必须为 false 。
3.4.4 长度检查： ​
  • @Size(max, min)：检查字段的 size 是否在 min 和 max 之间，可以是字符串、数组、集合、Map 等。
  • @Range可以配置min和max属性，同事显示最小值和最大值
3.4.5 日期检查： ​
  • @Future：被注释的元素必须是一个将来的日期。
  • @FutureOrPresent：判断日期是否是将来或现在日期。
  • @Past：检查该字段的日期是在过去。
  • @PastOrPresent：判断日期是否是过去或现在日期。
3.4.6 其它检查： ​
  • @Email：被注释的元素必须是电子邮箱地址。
  • @Pattern(value)：被注释的元素必须符合指定的正则表达式。（此value只能撞string类型） 注意：使用pattern可以不用添加@NotEmpty以及@NotBlank
3.4.7 @Valid和 @Validated的区别： ​
  • @Valid 注解，是 Bean Validation 所定义，可以添加在普通方法、构造方法、方法参数、方法返回、成员变量上，表示它们需要进行约束校验。
  • @Validated 注解，是 Spring Validation 锁定义，可以添加在类、方法参数、普通方法上，表示它们需要进行约束校验。同时，
  • @Validated 有 value 属性，支持分组校验。
3.4.8 Hibernate Validator 附加注解： ​

org.hibernate.validator.constraints

  • @Length：被注释的字符串的大小必须在指定的范围内
  • @Range：被注释的元素必须在合适的范围你
  • @URL：被注释的元素是一个URL

3.7 分组校验： ​

​ 如果不指定分组则默认的参数校验分组是在 Default 中。

java
public final <T> Set<ConstraintViolation<T>> validate(T object, Class<?>... groups) {}

​ 在 validate 校验方法中支持传入指定的分组。

新建分组方式：

​ 在类中新建一个标记，然后在注解中使用 group 属性进行指定：

java
public class User {
    // 分组标记：
    public interface Add{}
    public interface Update{}

    @NotNull(groups = Add.class)
    private String name;

    @NotNull(groups = Update.class)
    private int age;

    @Test
    public void test() {
        User user = new User();
        // 传入要检验的参数分组：
        List<String> valid = ValidationUtils.valid(user, Add.class);
        System.out.println(Arrays.toString(valid.toArray()));
    }
}
java
import javax.validation.ConstraintViolation;
import javax.validation.Validation;
import javax.validation.Validator;
import java.util.List;
import java.util.Set;
import java.util.stream.Collectors;

public class ValidationUtils {
    // 线程安全：
    private static Validator validator;

    static {
        validator = Validation.buildDefaultValidatorFactory().getValidator();
    }

    /**
     * 参数校验方法
     *
     * @param t 被校验的参数
     * @return List 校验信息
     * @param <T> 传入参数指定泛型
     */
    public static <T> List<String> valid(T t, Class<?>... group) {
        // 进行参数校验，如果检验没通过，返回的 validate 中，保存了校验信息：
        Set<ConstraintViolation<T>> validate = validator.validate(t, group);
        List<String> list = validate.stream().map(v -> "属性：" + v.getPropertyPath()
                        + "\t属性值：" + v.getInvalidValue()
                        + "\t提示信息：" + v.getMessage())
                .collect(Collectors.toList());

        return list;
    }
}

同一个属性上，通过分组可以实现不同的校验规则。
